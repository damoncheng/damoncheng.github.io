Start From Github

Word Explain

  • Issue : 一个Issue是一个需要注意或者注解的笔记。它可能是一个bug, 一个特征请求，一个问题或者许多另外的事情。在github上可以标记，搜索和分配Issues，从而使得管理一个active的工程更容易。

  • Branch :

    1. 当你有一个新的ideal的时候，你可以新建拷贝分支来实现。

    2. developers, writers, and designers可以使用不同分支来实现bug修复和特征工作，从而与master(product branch)分割开。当一个特征或者一个修复ready的适合，再合并到master。

  • Commit : 提交的集合是你工程的历史，每次提交都会关联一个消息，通过消息自己和其他人能够更好的理解这次提交所做的工作。

  • Pull Request : 提议某人拉取你的改变和请求，将你的贡献合并到他们的分支。增减内容的形式被叫做diff, 这个功能有特色的地方包含交流功能(discussion)，可以在最终merge前进行沟通进行调整好。

    note : 当你Pull Request的时候你可以在后面更上 Closes #32 来关闭指定的Issue。

  • Merge your Pull Request : 合并Pull Request到指定的分支。 如果合并的分支不再被使用，应该被删除掉。

  • Deloy : 在对Pull Request review后，将其部署到生产环境中进行验证，当出现问题时，可以roll it back。

Contributions

  • What counts as a contribution

    1. 提交到仓库的默认分支或者gh-pages分支
    2. 打开一个issue
    3. 提议一个pull request

    note : contributions仅仅统计过去一年的操作
